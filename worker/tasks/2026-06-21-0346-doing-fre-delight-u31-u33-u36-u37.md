- **U31(a) Boss Watch pill regression:** `BossWatchHeaderToggle` (App ~3880) reads `model.bossWatchPresentation` (App ~10908 → `BossWatchPresentation.resolve(isEnabled:)`, Core). That resolve does NOT consider whether a usable boss exists, so on a no-boss first run the pill renders green "Watch On". The calm header (`BossSelectorView`, App ~3915) already derives usability via `HeaderCalmPresentation` + `model.currentBossIsUsable` (App ~9994 → `ouroAgent(named:)?.isUsableAsBoss`). Fix: extend `BossWatchPresentation.resolve` to take `hasUsableBoss`; when false → `.hidden`-style presentation and the header view drops the pill.
- **U31(b) quiet-machine status line:** `HeaderView` (App ~3657) renders `model.summary.oneLineStatus` unconditionally. `WorkspaceSummary.oneLineStatus` (Core) returns `"0 running, nothing to recover"` when nothing's waiting + 0 running + 0 actionable recovery. Fix: a pure should-show/text derivation in Core (new `HeaderStatusLinePresentation`) — hide on a genuinely quiet machine (no waiting, 0 running, 0 actionable recovery); the boss prompt builder keeps the raw `oneLineStatus`.
- **U31(c) tray calm:** `WorkbenchMenuBarController.rebuildMenu` (App ~681-692) renders `"Boss: \(name)"` + `"TTFA · \(state.displayName) — \(headline)"` independently. Reuse `HeaderCalmPresentation.resolve` so no-boss reads calm (label "No boss yet", neutral TTFA "TTFA · off").
- **U33 two menus:** `RunningSessionHeaderControls` (App ~8938) renders the "Session Controls" menu (send-keys from `WorkbenchSurfacePolicy.sessionControls` advancedActions + Copy Launch / Open Dir) AND the in-header "More" menu (App ~8370) renders Ask Boss + Copy Launch + Open Dir + Edit/Duplicate/Move/Archive/Delete. Copy Launch / Open Dir duplicated. Fix: one Core seam (`SessionActionMenu`) laying out labelled sections; one SwiftUI overflow menu; primary Stop stays a sibling button.
- **U36 empty-state card:** `AgentHomeEmptyState` (App ~2783) renders an inert "Installed agents" card (bare HStacks, green/orange dot). `SidebarAgentRow` (App ~3141) wraps the same vocabulary in a real `Button(action: select)` + `.help(agent.detail)`. Fix: make the card rows real `Button`s calling `model.selectAgent(agent.name)`, keyboard-reachable, `.help(agent.detail)`; per-status dot color (3-way: ready green / disabled+missingConfig orange / invalidConfig red) + a human-readable reason for non-ready rows. Reuse `SidebarAgentRow` directly.
- **U37 palette:** `commandPaletteItems` (App ~11115). `for agent in ouroAgents` (App ~11563) emits one Select Agent per bundle incl. the boss, and any duplicate inventory record double-emits. Fix: a pure `WorkbenchCommandSection` grouping + a pure agent-select list builder (one per installed bundle de-duped by name, excluding the current boss). Rename "Set Up Workbench" → "Set up a boss" (App ~11156 + ~3718 menu).
