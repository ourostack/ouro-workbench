## Units

### U29 ✅ Get-or-create on the boss session-create path
- **Backlog:** [U29] — one `workbench_create_session` call provisions the workspace (validated per U14) and lands a terminal in it; strict must-already-exist stays the default.
- **Core seam (TDD):** `WorkbenchSessionGroupResolver` — pure resolution of `(groupValue, createIfMissing, rootPath)` against `WorkspaceState`:
  - existing UUID/unique-name match → `.existing(project)`
  - missing + `createIfMissing` + usable rootPath (validated via injected probe, reusing `WorkspaceRootValidation`) → `.create(name, expandedRootPath)`
  - missing + `createIfMissing` + bad/empty rootPath → `.invalid(message)`
  - missing + NOT `createIfMissing` → `.mustExist(message)` (today's strict error text)
  - nil/empty group → `.deferred` (app uses selected group)
- **MCP wiring (source-pinned):** `createSession` accepts `createGroupIfMissing: Bool` + uses the resolver. On `.create`, enqueue a `createGroup` action (validated rootPath) THEN the `createSession` action in one call; ack returns the resolved/created group name + requestIds. On `.existing`/`.deferred`, today's behavior. On `.mustExist`/`.invalid`, throw a clear tool error. Update tool description + inputSchema.
- **Acceptance:**
  - [x] A single create-session MCP call targets a named group and creates it (with a given root path) if missing.
  - [x] The call returns both the resolved group id/name and the new session requestId.
  - [x] Strict must-already-exist behavior remains the default for callers that pass a known group / omit the flag.
- **Tests:** `WorkbenchSessionGroupResolverTests` (Core) covers every resolver arm incl. tilde expansion + bad-path rejection. Source-pin the MCP wiring (handler builds createGroup+createSession requests). Build+test+coverage green.

Pure FS via injectable dir.
- **Core seam (TDD), part (b):** `WorkbenchReportBugRenderer` — `toolName = "workbench_report_bug"`, `toolDescription` (states text-yes / screenshot-no anonymization), and an `ack` builder. The app drains `.reportBug` and writes the bundle through the SAME `BugReportWriter` + `WorkbenchBugReportRedactor` the in-app reporter uses (live state needed → app-drain, not MCP-side).

The status store makes filed/URL survive relaunch. Confirm the boss-file path goes through `BugReportWriter` (same redactor) — pinned by test.
- **Acceptance:**
